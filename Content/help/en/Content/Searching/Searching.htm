Searching

Content Manager Web Client has three types of Searching:

Search Query - this allows users to search for Content Manager Records by typing in Search Method(s) and Parameter/Criteria(s).

NOTE: The Search Query search option requires users to know the Search Method syntax and the Boolean operators AND, OR and NOT, if they want to run advanced searches.
NOTE: When searching for Records that have a special character in them, e.g. a colon : the search query needs to be enclosed using quotes. For example, if the title of a Record is report:2014, then the search query would be title:"report:2007".

Search Editor - this enables users to build searches using search methods and criteria. Users can build simple searches or use advanced search operators such as Boolean operators, recursive searches, nested searches and grouped searches to build complex searches to find Content Manager items.

Search Form - this enables users to search for Content Manager items using predefined Search Methods. The Search Form dialogue is created by a Content Manager Administrator (see the Content Manager Help file for details).

Saved Search - this enables users to search for Content Manager objects using previously saved Search Methods and Clauses.  Users can save their own searches (see Creating a Saved Search) or they can be created by other Content Manager users who have the required permissions to create searches on behalf of other users.

Search History

When executing a search via the Search Query field, a history of the most recent 8 searches conducted using any of the searching options is available. This allows you to quickly use a recent search without having to rebuild it.

To remove a recent search, from the displayed list, click the X next to the search clause.
